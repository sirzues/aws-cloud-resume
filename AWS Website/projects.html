Automated IP Reputation Classification

Built an automated workflow that processes firewall IP addresses in controlled batches, classifies them as malicious or clean using reputation intelligence, and prepares results for reporting while respecting API limits and failure conditions.

Security Automation Cloud
Read more ▾

How it works

The workflow is designed to safely process large IP datasets under real-world constraints.

IP addresses are read once from a source spreadsheet and cached in memory to minimize repeated reads and avoid quota exhaustion. A batch controller processes two IPs at a time with a timed delay to comply with third-party API rate limits.

Before analysis, each IP passes through a validation gate to detect and skip private or invalid ranges, preventing unnecessary errors. Valid public IPs are sent to an external reputation service for analysis and classified as malicious or clean based on predefined criteria.

Results are merged and returned to the batch controller, allowing the pipeline to continue running regardless of mixed outcomes or partial failures.

Key design considerations

  • Rate-limited batch processing
  • API quota protection
  • Input validation before execution
  • Failure-tolerant looping and retry handling
  • Continuous execution without hard stops

Outcome

The workflow eliminates manual IP reputation checks, safely processes large datasets, and integrates cleanly into existing reporting workflows. It reduces manual classification effort by approximately 80% while maintaining accuracy and operational stability.

Organization-Wide AWS Security Alerting Pipeline

Built an organization-wide security alerting pipeline that captures high-value AWS security events across all accounts and delivers real-time alerts to a centralized inbox using native AWS services.

Cloud Security Centralized Logging Event-Driven Alerting
Read more ▾

What it does

The pipeline continuously monitors activity across all AWS accounts in an Organization and surfaces security-relevant events in near real time.

It collects organization-level CloudTrail events, filters for high-signal security actions (such as console logins and identity changes), transforms raw logs into readable alerts, and delivers them directly to email via SNS.

How it works

An organization-level CloudTrail trail captures management events from all member accounts and writes them to a centralized CloudWatch Log Group.

A CloudWatch Logs subscription filter applies keyword-based filtering and forwards only security-relevant events to a Lambda function. A custom filter pattern is used to reduce noise and ensure only high-value events are processed.

The Lambda function parses CloudTrail JSON logs, extracts key fields, builds a structured alert with a dynamic subject and message body, and publishes the result to an SNS topic.

SNS then delivers the alert to subscribed recipients, such as a security team inbox.

Key components

  • AWS Organizations – Centralized governance and organization-wide visibility
  • CloudTrail (Org Trail) – Captures management events from all accounts
  • CloudWatch Logs – Centralized log storage
  • Subscription Filter + Filter Pattern – Denoises logs and isolates high-value events
  • Lambda – Transforms raw logs into structured, readable alerts
  • SNS – Delivers alerts to email subscribers in real time

Outcome

  • Centralized visibility into security-relevant activity across all accounts
  • Real-time alerts without manual log review
  • Reduced noise compared to raw CloudTrail logs
  • Readable, actionable notifications delivered directly to inbox

Why it matters

In multi-account AWS environments, visibility is critical. This project demonstrates how native AWS services can be combined to build a scalable, organization-wide security monitoring pipeline that prioritizes signal over noise and delivers actionable insight without relying on third-party tooling.

Login Failure Burst Detection (AWS)

Built a real-time detection pipeline that identifies suspicious login failure bursts across an AWS environment and automatically sends alerts when predefined thresholds are exceeded.

Cloud Security Behavioral Detection Event-Driven Automation
Read more ▾

What it does

This system continuously monitors authentication activity and detects potential brute-force or credential misuse attempts by identifying repeated login failures for the same user within a short time window.

When suspicious patterns are detected, a detailed alert is generated and delivered automatically — no manual log review required.

How it works

CloudTrail management events are ingested via CloudWatch Logs and stored as failed login events in DynamoDB.

A scheduled detection Lambda, triggered by Amazon EventBridge, scans the event data and correlates login failures per user across a rolling five-minute window. When the failure count for a user exceeds a defined threshold, the Lambda generates a structured security alert.

The alert includes relevant context such as the username, source IP address, and timestamps, and is delivered via SNS directly to an inbox.

A second DynamoDB table tracks the last scan time, ensuring efficient processing without re-evaluating previously analyzed events.

Key components

  • DynamoDB (Events Table) – Stores failed login attempts
  • DynamoDB (State Table) – Tracks last scan time for efficient detection
  • Detection Lambda – Correlates login failures and applies detection logic
  • Amazon EventBridge – Triggers detection on a fixed schedule
  • SNS – Delivers alerts automatically
  • CloudTrail + CloudWatch Logs – Source of raw authentication events

Detection logic

  • Groups failed login attempts by user
  • Evaluates activity within a five-minute window
  • Triggers an alert when three or more failures occur for the same user
  • Sends contextual details including IP address, username, and timestamps

Outcome

  • Real-time visibility into suspicious login behavior
  • Automated detection without manual log analysis
  • Actionable alerts with clear context
  • Scalable pattern-based detection using native AWS services

Why it matters

Effective security monitoring isn’t just about collecting logs — it’s about correlating behavior and surfacing risk quickly. This project demonstrates how simple detection logic, when applied consistently and automatically, can provide meaningful security visibility across an AWS environment.

GuardDuty-Driven SSH Brute-Force Detection & Auto-Remediation

Built an automated security pipeline that detects SSH brute-force attempts using Amazon GuardDuty and automatically blocks the offending IP address by updating a network access control list (NACL) in real time.

Cloud Security Event-Driven Remediation Automated Incident Response
Read more ▾

What it does

This system monitors AWS GuardDuty findings for SSH brute-force activity against an EC2 instance in a public subnet and responds automatically.

When a threat is detected, it generates a clean, readable alert and applies immediate network-level remediation without manual intervention.

How it works

Amazon GuardDuty continuously analyzes VPC flow logs, CloudTrail events, and DNS logs to detect suspicious behavior. When an SSH brute-force finding is generated, an Amazon EventBridge rule matches the event pattern and routes it through the pipeline.

The finding is first sent to a Lambda function that parses and restructures the raw GuardDuty data into a concise, human-readable alert. This cleaned alert is then published to SNS and delivered via email.

For remediation, a second Lambda function is triggered by the same EventBridge rule. This function extracts the offending IP address from the GuardDuty finding and automatically updates the VPC’s Network ACL to block further access from that source.

As a result, detection, alerting, and remediation occur within seconds of the initial attack signal.

Key components

  • Amazon GuardDuty – Detects SSH brute-force activity
  • Amazon EventBridge – Routes security findings based on event patterns
  • Lambda (Alert Processing) – Parses and formats GuardDuty findings
  • Lambda (Remediation) – Updates the NACL with the malicious IP
  • Network ACL (NACL) – Enforces network-level blocking
  • SNS – Delivers clean security alerts via email

Outcome

  • Automated detection of SSH brute-force attempts
  • Immediate network-level blocking of malicious IPs
  • Clear, readable alerts instead of raw GuardDuty logs
  • Reduced response time with no manual intervention

Why it matters

Security automation is most effective when detection and response are tightly coupled. This project demonstrates how AWS native services can be combined to move from alerting to action, reducing exposure time and ensuring consistent, repeatable responses to common attack patterns.

Automated Personal Finance & Spending Insights Pipeline

Built an automation workflow that ingests transaction alerts from email, cleans and categorizes the data using AI, stores it in a structured format, and generates automated spending insights without manual tracking.

Process Automation AI-Assisted Workflows Orchestration
Read more ▾

What it does

This system replaces manual expense tracking by automatically processing transaction alerts as they arrive.

It ingests raw transaction notifications, categorizes and cleans the data using AI, stores the results in Google Sheets, and generates summarized insights about spending behavior and trends.

How it works

Transaction alert emails are ingested directly from Gmail and passed into an automation workflow.

AI is used to extract relevant fields, normalize transaction data, and categorize each entry consistently. Cleaned transactions are stored in Google Sheets, where duplication checks and formatting are handled automatically.

A second AI-driven step analyzes the accumulated data to identify spending patterns and generate summarized insights, which are then delivered automatically without manual reporting.

Key components

  • Gmail ingestion – Captures transaction alerts in real time
  • AI processing – Cleans, normalizes, and categorizes transaction data
  • Google Sheets – Structured storage and lightweight reporting
  • AI analysis layer – Generates spending insights and trends
  • n8n – Orchestrates the end-to-end workflow

Outcome

  • Eliminates manual expense tracking
  • Removes duplication and formatting overhead
  • Provides consistent categorization and insights
  • Saves hours of recurring administrative work

Why it matters

Automation doesn’t start with complex systems — it starts by removing friction from everyday processes. This project demonstrates how simple automation principles can be applied to personal workflows in the same way they are applied in larger systems, using orchestration, data normalization, and automated analysis to replace repetitive manual work.
